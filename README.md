# Foundation-of-computer-science-Task-3

## General Overview
```
- This project is based on a college system that manages student club memberships.
- The original data is stored in one big table (Student + Club + JoinDate together).
- This causes duplicate data and common database problems.
- The project fixes the data using normalization (1NF, 2NF, 3NF), creates an ER diagram, and writes SQL queries.
```

## Learning Objectives
```
- Identify data problems in an unnormalized table.
- Understand why normalization is needed.
- Convert a table into 1NF, 2NF, and 3NF.
- Create an ER diagram with PK, FK, and relationships.
- Write SQL queries (INSERT, SELECT, JOIN).
- Understand why JOIN is needed after normalization.
```
